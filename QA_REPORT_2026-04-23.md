# BoltBerry — Full QA Report

**Date:** 2026-04-23
**Branch:** `claude/full-qa-testing-6gOCb`
**Scope:** Feature logic flaws, usability/UX, performance & bottlenecks
**App version:** 0.20.17 (Electron 32 / React 18 / TypeScript 5 / SQLite)

This audit reviewed ~170 source files across `src/main`, `src/preload`, `src/renderer`

## 1. Feature Logic Flaws

### Critical

1. **Stale-closure on save-state timer**
   `src/renderer/stores/appStore.ts:13-20` — `savedTimer` is module-scoped and
   shared across `setSaving` / `setSaved` calls. Rapid consecutive saves can
   clear a later timer from an earlier handler, collapsing the "Saved" UI to
   "Idle" prematurely. Repro: two `setSaving()` within 500 ms followed by
   `setSaved()`.

2. **`warnedFull` leak in undo store**
   `src/renderer/stores/undoStore.ts:114-119` — `warnedFull` lives in closure;
   the reset lives inside a `setTimeout` that may never fire if the component
   unmounts, so the "undo stack full" toast never reappears for the rest of
   the session.

3. **Race condition / missing null-check in player bridge**
   `src/main/ipc/player-bridge.ts:22-28, 51-61` — `getDMWindow()?.webContents`
   is checked, but the subsequent `getPlayerWindow()?.webContents.send()` can
   fire on a destroyed `webContents` if the DM window closes mid-flight,
   crashing IPC.

### High

4. **Initiative effect-timer decrements at wrong moment**
   `src/renderer/stores/initiativeStore.ts:76-88` — Timers decrement on round
   wrap, but if combat starts mid-round the first `nextTurn()` wraps
   immediately and decrements early. Combat-effects expire one round early.

5. **`FOG_SAVE` handler lacks validation & error propagation**
   `src/main/ipc/fog-handlers.ts:34-52` — No size cap on the incoming data
   URL and no `try/catch` around the SQLite upsert. A bad renderer payload
   can exhaust memory or silently corrupt the fog snapshot.

6. **Fog desync between DM and Player on mode flip**
   `src/renderer/PlayerApp.tsx:232-241` — `onFogDelta` early-returns when
   canvas refs are null during mode transition, but the fog version counter
   still increments, leaving the two screens out of sync until a full
   resync.

7. **Off-by-one / OOB access in grid detection**
   `src/renderer/utils/gridDetect.ts:177, 206` — `findTopPeaks` loop skips
   entirely when `maxLag >= signal.length - 1`; `harmonicBonus` reads
   `signal[idx + d]` without bound check. Edge-case inputs yield wrong
   peaks / `undefined` arithmetic.

8. **Non-integer numeric coercion for integer columns**
   `src/main/ipc/token-handlers.ts:108, 137-139` — `coerceNumber` accepts
   `1.5` for `size`, `hp_current`, `hpMax`, `zIndex`. Silent data-type
   corruption in SQLite.

### Medium

9. **Import loses data from older exports**
   `src/__tests__/export-import-roundtrip.test.ts:47` vs.
   `src/main/ipc/export-import.ts:14` — `EXPORT_VERSION = 9` with no
   upgrade path for v8 archives (audio boards, grid customisation
   silently dropped).

10. **Path handling in compendium handler is permissive**
    `src/main/ipc/compendium-handlers.ts:147-150` — `COMPENDIUM_READ` accepts
    any `filePath` from renderer; `realpathSync` check runs after split.
    Mitigated by userData scope but should be narrowed to the compendium
    directory.

11. **Silent drawing-data loss on parse error**
    `src/main/ipc/drawing-handlers.ts:29-45` — `parsePoints` swallows JSON
    errors and returns an empty shape; a corrupted row survives in the DB
    but disappears from the canvas with no warning.

12. **`updateEntry` fails silently on unknown id**
    `src/renderer/stores/initiativeStore.ts:54-61` — If `find()` misses, the
    patch vanishes and the sort still runs. No log, no toast.

13. **Fog history cap off-by-one**
    `src/renderer/stores/fogStore.ts:41, 62` — `slice(-49)` before `push`
    leaves 51 entries, not 50. Same bug in `redo()`.

14. **`markerColor` not format-validated**
    `src/main/ipc/token-handlers.ts:120` — Any string stored verbatim;
    depending on the renderer path this can become an XSS sink.

15. **Character portrait paths not remapped on import**
    `src/main/ipc/export-import.ts:458` — `portraitPath` is exported but
    `remapPaths()` doesn't update it, so imported sheets show a broken
    portrait.

### Low

16. Redundant `[...entries]` spread before sort — `initiativeStore.ts:40`.
17. German-only toast text — `undoStore.ts:118`.
18. `ensureTokenVariantsSeeded` swallows errors — `compendium-handlers.ts:106-130`.
19. Character sheet updates outside an explicit transaction —
    `character-sheet-handlers.ts:76-119`.
20. Grid-colour default duplicated across schema, handlers, and export —
    risk of drift.

## 3. Performance & Bottlenecks

### Critical

54. **Full-roster IPC on token drag (10 Hz)**
    `src/renderer/components/canvas/TokenLayer.tsx:180-210` — every 100 ms
    the entire visible token array is filtered, mapped and broadcast.
    Saturates IPC on dense maps (50+ tokens).

55. **`broadcastTokens` called after every mutation**
    `TokenLayer.tsx:554, 577, 588, 761, 835, …` — HP changes, visibility
    toggles, status effects: each sends the full list. No diff / delta.

56. **LOS ray-casting is O(angles × walls) with no spatial index**
    `src/renderer/utils/losEngine.ts:99-127` — ~20 angles × 100+ walls =
    ~2 000 ray/segment tests per visibility-polygon build. Runs on every
    lit-token move.

57. **No token virtualisation**
    `TokenLayer.tsx:832-867` — every token on the map always becomes a
    Konva node with hit-test + listeners, regardless of viewport. Konva
    hit-tests become O(n) on pan/zoom.

### High

58. Over-subscribed Zustand selectors re-render all `TokenNode`s on any
    store change — `TokenLayer.tsx:68-84`.
59. `toDataURL('image/png')` on two multi-MB canvases during fog undo/redo
    blocks the main thread — `FogLayer.tsx:223-225, 273-274, 309-310`.
60. Canvas 2D contexts re-acquired on every fog op instead of cached —
    `FogLayer.tsx:191, 203, 258`.
61. `sortedTokens` re-clones and re-sorts the full token list on every
    mutation — `TokenLayer.tsx:175`.
62. `useImage` LRU evicts Map entries without revoking object URLs —
    leaks ~50 MB over long sessions — `src/renderer/hooks/useImage.ts:6-16`.
63. Fog deltas broadcast unconditionally even with no player window —
    `FogLayer.tsx:670-706`.
64. Bestiary search re-filters + re-sorts 263 rows on every keystroke, no
    debounce — `components/bestiary/MonstersTab.tsx:68-90`.
65. `MapLayer` recreates `Konva.Image` for previously-loaded maps —
    `components/canvas/MapLayer.tsx:24-40, 49`.

### Medium

66. Fog undo rebuilds by replaying all ≤50 history ops instead of applying
    inverse deltas — `FogLayer.tsx:197-226`.
67. `[...tokens].sort(...)` allocates/sorts regardless of whether order
    changed — `TokenLayer.tsx:175`.
68. Grid FFT allocates a fresh `Float64Array` per row/column scan —
    `utils/gridDetect.ts:56-70, 238-262`.
69. Pointer auto-hide `setTimeout` not tracked for cleanup —
    `PlayerApp.tsx:209`.
70. Ray-casting lacks early-exit for opaque occluders —
    `losEngine.ts:116-119`.
71. Four canvases re-allocated per map load with no pooling —
    `FogLayer.tsx:51-101`.
72. `useAutoSave` writes `touchLastOpened` every 60 s even when idle —
    `hooks/useAutoSave.ts:10-13`.
73. Module-scoped `saveTimer` in `FogLayer` is a hot-reload hazard —
    `FogLayer.tsx:649, 692`.

### Low

74. Redundant filter+map allocations in `broadcastTokensFromCanvas` —
    `canvas/CanvasArea.tsx:44-58`.
75. Token name/HP `<input>` edits un-debounced (Konva re-renders per

76. Status-effect toggle uses `filter()` instead of `Set` —

77. PDF render task cancel is soft (non-eager) —

## Top Recommended Fixes (priority order)

1. **Token sync redesign** — move to per-token delta IPC; batch HP / status /
   visibility mutations; cap drag broadcasts via `requestAnimationFrame`
   instead of 100 ms wall-clock throttle. *(→ #54, #55, #74)*
2. **LOS / wall geometry index** — add BVH or uniform grid so ray-casts
   become O(log n) or near-constant. *(→ #56, #70)*
3. **Virtualise the token layer** — cull nodes outside viewport; skip
   hidden-to-players tokens from the player broadcast list. *(→ #57, #61, #67)*
4. **Repair module-scoped timers / flags** — move `savedTimer`,
   `warnedFull`, `saveTimer` into store state or refs. *(→ #1, #2, #73)*
5. **Harden IPC boundary** — size-limit fog bitmaps, validate
   `markerColor`, integer-coerce integer columns, confine compendium paths.
   *(→ #5, #10, #8, #14)*
6. **Fix internationalisation** — flip `fallbackLng` to `en`; migrate
   hardcoded strings in `SessionStartModal`, `SetupWizard`,
   `ShortcutOverlay` and `undoStore` into `i18n` keys; add a `check:i18n`
   test for symmetric coverage. *(→ #17, #21-#24)*
7. **Accessibility pass** — trap focus in modals, associate labels with
   inputs, add `role="tab"` / `aria-selected`, surface errors inline, add
   non-colour HP cue. *(→ #27, #28, #30, #33, #34)*
8. **Fog pipeline rewrite** — cache 2D contexts, replace `toDataURL`
   snapshots with inverse deltas or `ImageData` blobs, pool canvases across
   maps, gate broadcasts on player window presence. *(→ #59-#63, #66, #71)*
9. **Image-cache lifecycle** — revoke blob URLs on eviction; add retry on
   error. *(→ #62, #80)*
10. **Initiative + import correctness** — fix round-boundary decrement,
    remap portrait paths, add version migrations for exports. *(→ #4, #9,
    #15)*
